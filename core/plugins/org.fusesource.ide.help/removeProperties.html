Remove Properties

Table of Contents

Usage
Properties

Removes from the message exchange all properties matching the specified pattern or patterns, but preserves those properties that match the pattern or patterns to exclude, when specified

Usage

The Remove Properties pattern removes all properties matching the specified patterns from the message exchange, except those matching the patterns to exclude, if specified.

The Remove Properties pattern can appear anywhere in the route body.

Properties

Table 109, “Remove Properties Properties” describes the properties you can specify using the properties editor.

Table 109. Remove Properties Properties

NameDescription

Pattern

Specifies the property name pattern or patterns of the properties to remove from the message exchange.

Description

Specifies a text description for the node. This description is included in the generated XML file, but it is informational only. It is not used by Apache Camel.

Exclude Pattern

Specifies the property name pattern or patterns of the properties to preserve in the message exchange.

Id

Specifies a unique identifier for the endpoint.

The tooling automatically generates an id for a node when it is created, but you can remove that id or replace it with your own. The Camel debugger requires all nodes with a breakpoint set to have a unique id.

You can use the id to refer to endpoints in your Camel XML file.
